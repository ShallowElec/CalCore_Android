# Calcore Android 架构设计文档

> 本文件是 Android 端的详细架构蓝图，涵盖 UI 架构、渲染管线、模块划分与 Material You 适配策略。
> 状态：**草案**（与用户讨论后固化）

---

## 1. 设计目标

1. **Material You 原生融合**: 利用 Android 12+ 的 Dynamic Color，让计算器这个「硬核工具」也能融入用户的个性化系统美学。
2. **双模式架构**: 标准计算器模式与图形计算模式共享底层引擎，但 UI 布局与渲染路径完全不同。
3. **60 FPS 可视化**: 底层原理动画和图形渲染都必须在主流设备上稳定 60 FPS。
4. **教育级可扩展性**: 可视化层级（L1-L8）、架构模型（RISC-V/ARM64/x86-64）、数学对象类型均为插件化预留。

## 3. Material You 适配策略

### 3.1 主题架构（三套配色）

```kotlin
// ui/theme/Theme.kt
enum class AppTheme {
    SYSTEM_DYNAMIC,   // 跟随 Material You 壁纸取色
    CALCORE_DARK,     // 终端绿：#0a0a0a + #00ff41
    CALCORE_LIGHT     // 浅蓝风：#FFFFFF + 浅蓝/蓝色系
}

@Composable
fun CalcoreTheme(
    appTheme: AppTheme = AppTheme.SYSTEM_DYNAMIC,
    darkTheme: Boolean = isSystemInDarkTheme(),
    content: @Composable () -> Unit
) {
    val colorScheme = when (appTheme) {
        AppTheme.SYSTEM_DYNAMIC -> {
            if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.S) {
                val context = LocalContext.current
                if (darkTheme) dynamicDarkColorScheme(context)
                else dynamicLightColorScheme(context)
            } else {
                CalcoreDarkScheme  // API < 31 回退
            }
        }
        AppTheme.CALCORE_DARK -> CalcoreDarkScheme
        AppTheme.CALCORE_LIGHT -> CalcoreLightScheme
    }

    MaterialTheme(
        colorScheme = colorScheme,
        typography = CalcoreTypography,
        shapes = CalcoreShapes,
        content = content
    )
}
```

### 3.3 品牌色与动态色的融合策略

| 场景 | 策略 |
|---|---|
| **App 主背景** | 由当前配色方案决定：系统动态色跟随壁纸；终端绿用 `#0a0a0a`；浅蓝白用纯白 `#FFFFFF`。 |
| **可视化舞台背景** | 固定 `#0a0a0a`（内容画布），不受配色方案影响，保持底层沉浸感。浅蓝白模式下可视化区仍保持深色，形成「暗黑舞台 + 明亮 UI」的对比美学。 |
| **数据流/强调色** | UI 控件使用 `MaterialTheme.colorScheme.primary`；可视化区保留科技绿 `#00ff41` 作为数据流语义色（不受主题切换影响）。 |
| **警告/溢出** | UI 层使用 `MaterialTheme.colorScheme.error`；可视化区保留琥珀色 `#ffa657` 作为进位指示语义色。 |

### 3.3 类型系统 (Typography)

- **等宽字体**: 地址、寄存器名、二进制位格使用 `FontFamily.Monospace`。
- **数学公式**: 集成 `androidx.compose.ui.text` 的 `SpanStyle` 实现上标/下标；长期预留 MathML / LaTeX 渲染接入点。
- **层级**: `displayLarge` 用于计算结果；`headlineMedium` 用于模式标题；`bodyMedium` 用于寄存器标签；`labelSmall` 用于内存地址。

## 5. UI 布局设计

### 5.1 标准模式（手机竖屏）

```
┌─────────────────────────────┐
│ [TopAppBar] 模式选择 Chip   │  <- 滚动选择：标准 | 科学 | 程序员 | 日期
├─────────────────────────────┤
│                             │
│      可视化舞台 (可折叠)     │  <- Compose Canvas，默认半屏
│      [内存/寄存器/ALU]      │     折叠后仅留一条指示条
│                             │
├─────────────────────────────┤
│      显示屏                 │  <- CalcoreDisplay，等宽字体
│      当前表达式 + 结果       │
├─────────────────────────────┤
│  [历史] [MR] [MC] [M+] [M-] │  <- 功能键行
├─────────────────────────────┤
│  7  8  9  ÷                 │  <- 数字键盘

**Material You 适配**:
- 按键使用 `FilledTonalButton`（中等强调），非主色按钮，避免喧宾夺主。
- 运算符键（`÷×-+`）使用 `FilledButton`（primary 容器色）。
- 等号 `=` 使用 `FilledButton`（primary 色），最大视觉权重。
- 可视化舞台折叠/展开动画使用 `AnimatedContentSize` + `animateContentSize()`。

```

│                        │                          │

│   [显示屏]              │    [Canvas]              │

## 6. 渲染管线设计

### 6.1 2D 可视化舞台（Compose Canvas）

```
用户按键 → ViewModel 生成 AnimationEvent
                ↓
    VisualizationViewModel 接收 Event
                ↓
    查询 Engine 生成 AnimationScript (指令序列)
                ↓
    状态更新：uiState = uiState.copy(
                bitGrid = newBits,
                stackFrames = newStack,
                dataFlowPaths = newPaths,
                timelineDuration = totalMs
              )
                ↓
    Compose 重组 → VisualizationStage Composable
                ↓
    Canvas(modifier = Modifier.fillMaxSize()) {
        drawBitGrid(bitGrid)
        drawStack(stackFrames)
        drawDataFlow(dataFlowPaths, progress)
        drawTimeline(currentTime)
    }
                ↓
    LaunchedEffect(animating) {
        while (animating) {
            currentTime += 16ms
            delay(16)
        }
    }
```

**性能策略**:
- `Canvas` 的 `draw` lambda 在后台线程的 Skia 中执行，但状态更新在主线程。
- 使用 `remember { Path() }` 缓存静态路径（如内存网格线），避免每帧重建。
- 位格翻转动画使用 `Animatable<Float>` 控制 alpha/scale，而非每帧重算 Path。
- 大型内存网格（如 64 位 × 多层）启用裁剪：只绘制视口内单元格。

### 6.2 3D 图形渲染（OpenGL ES）

```
用户输入表达式 → Engine 采样生成 VertexBuffer (FloatArray)
                        ↓
    GraphingViewModel 持有 VertexBuffer 引用
                        ↓
    GLSurfaceView.Renderer.onDrawFrame()
        ↓
    GLES20.glUseProgram(shaderProgram)
    GLES20.glUniformMatrix4fv(mvpMatrix, …)
    GLES20.glBindBuffer(GL_ARRAY_BUFFER, vboId)
    GLES20.glDrawArrays(GL_TRIANGLES, 0, vertexCount)
        ↓
    双缓冲 → SurfaceFlinger 合成
```

**交互同步**:
- 相机控制（旋转/平移/缩放）在 Compose 层通过 `pointerInput` 捕获手势，生成 `CameraState`。
- `CameraState` 通过 `StateFlow` 流向 `GraphRenderer`，在 `onDrawFrame` 中读取最新的 MVP 矩阵。
- 参数滑块变化 → 重新采样 → 更新 VBO → 请求 `requestRender()`。

### 6.3 动画时间轴架构

```kotlin
// domain/model/AnimationTimeline.kt
data class AnimationTimeline(
    val events: List<AnimationEvent>,      // 按时间排序的事件
    val totalDurationMs: Long,
    val currentTimeMs: Long = 0L,
    val isPlaying: Boolean = false,
    val speedMultiplier: Float = 1.0f
)

// presentation/visualization/VisualizationViewModel.kt
class VisualizationViewModel : ViewModel() {
    private val _timeline = MutableStateFlow(AnimationTimeline())
    val timeline: StateFlow<AnimationTimeline> = _timeline.asStateFlow()

    private var animationJob: Job? = null

    fun play() {
        animationJob?.cancel()
        animationJob = viewModelScope.launch {
            val start = SystemClock.elapsedRealtime() - _timeline.value.currentTimeMs
            while (isActive && _timeline.value.currentTimeMs < _timeline.value.totalDurationMs) {
                val elapsed = (SystemClock.elapsedRealtime() - start) * _timeline.value.speedMultiplier
                _timeline.update { it.copy(currentTimeMs = elapsed.toLong().coerceAtMost(it.totalDurationMs)) }
                delay(16) // ~60 FPS
            }
        }
    }

    fun scrubTo(timeMs: Long) { /* … */ }
    fun pause() { animationJob?.cancel() }
}
```

### 8.1 计算器模式状态机

```

        数字/小数点 →│   IDLE       │←── 清除/重置
        运算符      →│ (等待输入)   │
                    └──────┬───────┘
                           │ 数字输入
                           ▼
                    ┌──────────────┐
        运算符      →│  INPUTTING   │←── 退格
        等号        →│ (输入中)     │
                    └──────┬───────┘
                           │ 等号
                           ▼
                    ┌──────────────┐
        等号(重播)  →│  EVALUATED   │←── 模式切换
        新数字      →│ (已求值)     │     (重置为 IDLE)
        运算符      →│              │
                    └──────────────┘
```

每个状态转移都触发 `VisualizationEngine` 生成对应层级的动画脚本。

                    ┌──────────────┐

## 9. 性能预算

| 指标 | 目标 | 实现策略 |
|---|---|---|
| 按键 → 首帧动画 | ≤ 100ms | 预计算 AnimationScript，延迟 16ms 启动 |
| 动画帧率 | 60 FPS | `delay(16)` + 裁剪绘制区域 |
| 2D 函数采样+渲染 | ≤ 16ms | 自适应步长 + 后台 Coroutine 采样 |
| 3D 网格渲染 (10k 顶点) | ≤ 16ms | VBO + 视锥裁剪 + LOD |
| 内存峰值 | ≤ 256MB | 大网格流式加载；历史记录限制 1000 条 |
| APK 体积 | ≤ 50MB | 按需加载着色器；资源压缩 |
