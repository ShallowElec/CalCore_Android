## 1. 产品概述

**Calcore** 是一款面向开发者、计算机科学教育者、数学研究者及硬核技术爱好者的高级可视化计算器。它在外观上提供与传统科学/程序员计算器完全一致的操作体验（标准、科学、程序员、日期计算等模式），但其核心差异化在于：**每一次用户交互——无论是输入一个数字、按下运算符、移动光标还是触发等号——都会在主界面旁侧或下侧实时展开一幅“计算本质”的底层原理动画。**

更进一步，Calcore 突破传统计算器的数值边界，提供**强大的图形计算模式**：用户可以自由输入数学表达式，在 2D/3D 坐标系中实时渲染函数图像、几何点线面、参数曲面乃至抽象流形；通过参数滑块驱动动画，直观观察数学对象的演化过程；并可将任意求值过程展开为逐步计算的可视化链路。

Calcore 不只是一台计算器，它是一个**可交互的数学与计算机体系结构教学引擎**。用户看到的不再是黑盒结果，而是从数学表达式解析、坐标映射、几何光栅化，到底层的布尔代数、位运算、寄存器分配、内存寻址、堆栈帧构建的完整因果链条。

## 3. 功能需求

### 3.1 计算功能矩阵（覆盖全部模式）

| 模式 | 功能范围 | 备注 |
|------|---------|------|
| **标准** | 四则运算、百分比、倒数、平方根、记忆键 (M+/M-/MS/MR/MC) | 基础模式 |
| **科学** | 三角函数 (sin/cos/tan/反函数)、对数 (ln/log)、指数 (e^x/10^x)、阶乘 (n!)、幂运算 (x^y)、π、常数 e、角度/弧度/梯度切换 | 支持 64-bit 浮点精度 |
| **程序员** | 进制切换 (Bin/Oct/Dec/Hex)、位运算 (AND/OR/XOR/NOT/Shift)、字节/字/双字/四字视图、逻辑/算术移位、1's/2's 补码显示 | 核心模式，可视化重点 |
| **日期计算** | 两日期间隔、日期加减（天/月/年）、工作日计算 | 展示时间戳底层转换 |
| **单位换算** | 货币、体积、长度、重量、温度、能量、面积、速度、时间、功率、数据、压力、角度 | 展示浮点乘除与查表过程 |
| **图形** | 2D/3D 函数图像、参数方程、隐函数、点/线/面/流形、不等式、向量场、参数动画 | **本版本新增核心模式** |
| **内存与历史** | 历史记录栈、内存寄存器可视化、表达式回溯、图形表达式列表 | 与底层动画联动 |

### 3.2 核心差异化一：实时底层原理可视化引擎

#### 3.2.1 可视化触发时机
- **数字输入**：每按一个数字键，动画区展示该数字从 ASCII/Unicode → 二进制 → 寄存器/内存缓冲区的写入过程。
- **运算符输入**：展示操作符优先级解析、表达式树（AST）节点生成、中缀转后缀（调度场算法）的堆栈变化。
- **等号执行**：完整展开从表达式求值到最终结果的**全链路动画**。
- **光标移动**：展示内存地址指针偏移、缓冲区索引变化。
- **模式切换**：展示寄存器位宽变化（32-bit → 64-bit）、寻址空间切换。

#### 3.2.2 可视化内容分层（从硅到屏幕）

| 层级 | 展示内容 | 动画表现形式 |
|------|---------|-------------|
| **L1: 布尔代数** | 逻辑门 (AND/OR/XOR/NOT) 对每一位的运算 | 绿色信号流在逻辑门网格中流动 |
| **L2: 数值表示** | int / float / double 的 IEEE 754 / 补码二进制展开 | 位格翻转动画，符号位/指数位/尾数位高亮 |
| **L3: 寄存器与ALU** | 64-bit 寄存器组 (RISC-V/ARM64/x86-64 抽象模型) | 寄存器槽位高亮，数据搬运路径 |
| **L4: 内存布局** | 栈区 (Stack)、堆区 (Heap)、数据段、常量池 | 绿色实心方块（已分配）与空心方块（空闲/指针）构成的内存网格 |
| **L5: 数据结构** | 表达式解析用的链表、操作数数组、运算符栈 | 链表节点用绿色方块串联，指针用箭头/连线动画 |
| **L6: 指针与寻址** | 内存地址（64-bit 虚拟地址）、段偏移、指针解引用 | 地址总线动画，指针段位移拼接（Segment + Offset） |
| **L7: 指令集抽象** | 模拟 RISC-V / ARM64 / x86-64 的指令执行流 | 指令取指→译码→执行→写回的流水线动画 |
| **L8: 结果回显** | 计算结果从寄存器 → 内存缓冲区 → 显示驱动 | 数据流汇聚到显示区域 |

#### 3.2.3 关键动画规范

- **链表与内存语义**：
  - **绿色实心方块**：表示已分配的内存单元 / 链表节点 / 栈帧中的实际数据。
  - **绿色空心方块**：表示指针、引用、空闲槽位、预分配但未写入的地址。
  - **连线动画**：指针赋值时，连线从源地址“生长”到目标地址；链表插入时，节点方块滑动并重新接线。

- **堆栈搬运过程**：
  - 栈帧以垂直方向堆叠（自上而下或自下而上生长，依架构而定）。
  - 压栈 (PUSH)：绿色实心方块从寄存器区滑入栈顶，栈指针 (SP/RSP/ESP) 箭头下移。
  - 弹栈 (POP)：方块从栈顶滑出至寄存器/ALU，栈指针箭头上移。
  - 函数调用（如阶乘递归）：展示多层栈帧的叠加与销毁。

- **指针段位移拼接**：
  - 当涉及内存寻址时，将 64-bit 地址拆分为**段选择子 / 基址 / 偏移量**（依架构抽象）。
  - 动画展示段寄存器与偏移寄存器的值在 ALU 中物理拼接成完整地址的过程，地址总线以绿色流光表示。

- **光标与地址映射**：
  - 用户输入光标在数字串中的位置，映射到内存缓冲区数组的索引。
  - 光标移动时，一个“读取头”在内存网格中同步移动，高亮当前字符所在的内存地址。

- **64-bit 计算支持**：
  - 所有数值运算默认以 64-bit 精度进行。
  - 可视化区始终展示 64 个位格（8×8 或 1×64 布局），根据模式折叠/展开（如程序员模式 Hex 视图展示 16 组 4-bit 块）。

- **多架构模拟切换**：
  - 用户可在设置中选择 **RISC-V / ARM64 / x86-64** 抽象模型。
  - 差异仅体现在寄存器命名、栈生长方向、寻址方式、指令助记符上，动画引擎统一调度。

## 4. 图形计算模式（Graphing Mode）

### 4.1 模式概述

**图形模式**是 Calcore 的核心扩展，允许用户通过声明式表达式在 2D 或 3D 坐标系中创建、组合、分析数学对象。该模式不仅提供静态图像渲染，更强调**动态参数驱动**与**计算过程可视化**，使用户能够“看见”数学表达式如何被解析、采样、映射为屏幕像素，乃至底层数值如何流经寄存器与浮点单元。

对标参考：Windows 11 计算器 Graphing 模式 / MacOS Grapher / Desmos / GeoGebra 3D / Mathematica Plot。

### 4.2 表达式输入与管理

#### 4.2.1 表达式类型系统

| 类型 | 数学形式 | 维度 | 示例 |
|------|---------|------|------|
| **显函数** | `y = f(x)` 或 `z = f(x,y)` | 2D / 3D | `y = x^2 + 3x - 2`, `z = sin(x) * cos(y)` |
| **参数曲线** | `(x(t), y(t))` 或 `(x(t), y(t), z(t))` | 2D / 3D | `(cos(t), sin(t))`, `(t, t^2, t^3)` |
| **参数曲面** | `(x(u,v), y(u,v), z(u,v))` | 3D | `(cos(u)*sin(v), sin(u)*sin(v), cos(v))` |
| **隐函数** | `F(x,y) = 0` 或 `F(x,y,z) = 0` | 2D / 3D | `x^2 + y^2 = 25`, `x^2 + y^2 + z^2 = 1` |
| **不等式** | `y > f(x)`, `F(x,y) <= 0` | 2D / 3D | `y > x^2`, `x^2 + y^2 <= 1` |
| **离散点集** | `{(x₁,y₁), (x₂,y₂), ...}` | 2D / 3D | `(1,2), (3,4), (5,6)` |
| **向量场** | `F(x,y) = <P(x,y), Q(x,y)>` | 2D / 3D | `<y, -x>` (旋转场) |
| **流形（参数化）** | `φ: U ⊂ ℝⁿ → ℝᵐ` (n < m) | 2D / 3D | 局部参数化的环面、克莱因瓶片段 |

#### 4.2.2 坐标系支持

| 坐标系 | 适用维度 | 表达式形式 | 备注 |
|--------|---------|-----------|------|
| **笛卡尔 (Cartesian)** | 2D / 3D | `y=f(x)`, `(x,y,z)` | 默认坐标系 |
| **极坐标 (Polar)** | 2D | `r = f(θ)` | 角度单位可切换弧度/度 |
| **柱坐标 (Cylindrical)** | 3D | `r = f(θ,z)` | `(r, θ, z)` |
| **球坐标 (Spherical)** | 3D | `ρ = f(θ,φ)` | `(ρ, θ, φ)` |

#### 4.2.3 表达式列表 UI

- 每条表达式占据一行，左侧带颜色标识（可自定义）。
- 右侧提供可见性切换、删除、复制按钮。
- 支持文件夹/图层式分组管理。
- 表达式输入采用类 LaTeX 的实时排版（如输入 `x^2` 自动渲染为上标）。
- 自动补全：函数名、常量、已定义变量。

### 4.3 2D/3D 坐标系渲染引擎

#### 4.3.1 渲染管线

```
表达式输入 → 词法/语法分析 (AST) → 语义分析 → 采样策略选择 → 离散化/网格化
    → 顶点/几何生成 → 着色/光照 → 屏幕空间映射 → 像素输出
```

每一步均可在可视化面板中展开动画演示。

#### 4.3.2 采样与离散化策略

| 对象类型 | 采样方法 | 数学原理 |
|---------|---------|---------|
| **显函数 y=f(x)** | 自适应步长采样 | 基于导数 `f'(x)` 的曲率估计，在变化剧烈处加密采样点 |
| **参数曲线** | 等参数间隔 + 弧长自适应 | `s(t) = ∫‖r'(τ)‖dτ`，保持屏幕空间线密度均匀 |
| **参数曲面** | UV 均匀网格 + LOD | 视距决定网格细分级别（类似地形 LOD） |
| **隐函数** | Marching Squares (2D) / Marching Cubes (3D) | 基于连续函数的中间值定理，在网格边上找零交叉点 |
| **不等式区域** | 像素级覆盖测试 + 边界提取 | 对屏幕像素逐点计算，或使用扫描线填充 |

#### 4.3.3 关键数学对象渲染规范

- **点 (Point)**：
  - 2D：以 `(x, y)` 坐标渲染为带标签的标记点，支持多种形状（圆点、叉、方块）。
  - 3D：渲染为 billboard 或小球体，带深度遮挡与透视缩放。
  - 悬停显示精确坐标值。

- **线 (Line / Curve)**：
  - 2D 直线：由两点式、点斜式、一般式 `Ax + By + C = 0` 定义。
  - 2D 曲线：参数方程或显函数，支持切线/法线向量动态展示。
  - 3D 曲线：管状渲染（tube radius 可调）或纯线框，支持 Frenet 标架（切向量 T、法向量 N、副法向量 B）动画。

- **面 (Surface)**：
  - 显式曲面 `z = f(x,y)`：生成三角网格，支持法线贴图与光照模型（Phong/Blinn-Phong）。
  - 参数曲面：UV 线框 + 实体表面双模式切换。
  - 支持半透明、双面渲染、剖面切割（Clipping Plane）。

- **流形 (Manifold)**：
  - 定义：局部同胚于 ℝⁿ 的拓扑空间，在 Calcore 中通过**局部参数化图卡 (Atlas of Charts)** 可视化。
  - 渲染方式：
    - 对可整体参数化的流形（如球面、环面），直接使用参数曲面渲染。
    - 对不可整体参数化的流形（如克莱因瓶、实射影平面），展示**多个参数化图卡的拼接**，用不同颜色区分图卡边界（Transition Map）。
    - 展示切空间 (Tangent Space)：在流形上某点处渲染切平面/切向量。
    - 展示法向量场 (Normal Bundle)：对可定向流形，渲染连续的单位法向量。
  - 示例对象：
    - 2D 流形：球面 S²、环面 T²、莫比乌斯带、克莱因瓶。
    - 曲线流形：椭圆、李萨如曲线、摆线、螺线。

### 4.4 参数驱动与动画系统

#### 4.4.1 参数滑块 (Parameter Slider)

- 在表达式中定义参数，如 `a = 1`, `b = 0.5`，自动在侧边栏生成滑块控件。
- 滑块支持范围、步长、动画播放（正向/反向/循环）设置。
- 拖动滑块时，图形实时重绘（目标 60 FPS）。

#### 4.4.2 轨迹与追踪 (Trace)

- 对参数方程，可开启**轨迹模式**：保留历史帧的残影，展示参数变化时曲线的运动轨迹。
- 可追踪曲线上特定点，展示其坐标、速度向量、加速度向量的实时变化。

#### 4.4.3 时间轴动画

- 以时间 `t` 为隐含参数，驱动整个场景演化。
- 支持关键帧插值：用户在 t₀、t₁、t₂ 设置参数值，引擎自动平滑过渡。
- 动画可导出为视频（MP4/WebM）或 GIF。

### 4.5 计算过程可视化（Graphing 模式专属）

当用户在图形模式下执行**求值**（如在函数上某点计算值、求根、求导、求积分）时，右侧可视化舞台展示数学计算链路：

| 计算步骤 | 可视化内容 | 示例 |
|---------|-----------|------|
| **表达式解析** | AST 树从原始文本“生长”出来，节点高亮 | `sin(x)^2 + cos(x)^2` → 根节点 `+`，左右子树分别为 `^` |
| **变量代入** | 参数值流入叶子节点，向上传播 | x = π/4 代入后，sin 节点显示 `√2/2` |
| **逐层求值** | 从叶子到根，每层节点的计算结果以数据流形式汇聚到父节点 | `^` 节点接收 `√2/2`，输出 `1/2` |
| **数值近似** | 展示浮点运算的 IEEE 754 表示与舍入过程 | 1/3 在 float/double 中的二进制展开 |
| **几何映射** | 将数值结果映射到坐标系中的像素位置 | 计算 (x,y) → 屏幕 (u,v) 的矩阵变换动画 |
| **采样过程** | 在函数图像上展示采样点的生成与连接 | 从左到右，点逐个计算并以线段连接 |
| **积分近似** | 黎曼和、梯形法、辛普森法的矩形/梯形逐个累加动画 | `∫[a,b] f(x)dx` 的分割与求和过程 |
| **微分几何** | 切向量、曲率、挠率的实时计算与可视化 | 曲线某点处 osculating circle 的构造动画 |

## 5. 用户体验与界面设计

### 5.1 布局架构

#### 5.1.1 标准模式布局（计算器模式）

采用 **"双窗格沉浸式布局"**：

```
┌─────────────────────────────────────────────┐
│  [菜单]  标准 | 科学 | 程序员 | 日期 | ...    │
├──────────────────┬──────────────────────────┤
│                  │                          │
│   传统计算器UI    │    底层原理可视化舞台     │
│   (输入/按键/显示) │   (Manim级数学动画)      │
│                  │                          │
│   [显示屏]        │   [内存布局 / 寄存器]     │
│   [数字键盘]      │   [堆栈 / 链表 / 指针]    │
│   [功能键]        │   [ALU / 数据流]         │
│                  │                          │
├──────────────────┴──────────────────────────┤
│  [时间轴 scrubber]  [架构切换]  [层级过滤]     │
└─────────────────────────────────────────────┘
```

- **左侧/下方（依屏幕比例）**：传统计算器操作区，保证零学习成本。
- **右侧/上方**：可视化舞台，默认展开，可收起/全屏。
- **底部控制条**：动画时间轴（可回放、暂停、调速）、当前模拟架构标签、可视化层级过滤器（L1-L8 开关）。

#### 5.1.2 图形模式布局

采用 **"表达式 + 视口 + 可视化" 三栏布局**：

```
┌─────────────────────────────────────────────────────────────┐
│  [菜单]  标准 | 科学 | 程序员 | 日期 | 图形 | ...            │
├──────────────┬──────────────────────────────┬───────────────┤
│              │                              │               │
│  表达式列表   │      2D/3D 坐标系视口        │  计算过程可视化 │
│  + 参数控制   │                              │  / 底层原理    │
│              │   [网格 / 坐标轴 / 函数图像]   │               │
│  [y1 = x^2]  │   [旋转/平移/缩放控制]        │  [AST树 /      │
│  [y2 = 2x+1] │                              │   采样过程 /   │
│  [参数a滑块]  │                              │   数值流]      │
│              │                              │               │
├──────────────┴──────────────────────────────┴───────────────┤
│  [表达式输入栏]  [坐标系切换]  [工具: 求根/导数/积分/追踪]    │
└─────────────────────────────────────────────────────────────┘
```

- **左侧面板**：表达式列表、图层管理、参数滑块。
- **中央视口**：2D/3D 渲染区，支持鼠标/触摸交互（平移、缩放、旋转）。
- **右侧面板**：计算过程可视化（可切换为底层原理可视化 L1-L8）。
- **底部工具栏**：表达式输入、坐标系选择、分析工具（求根、极值、交点、导数、积分）。

### 5.2 交互范式

| 场景 | 交互设计 |
|------|---------|
| **实时跟随** | 用户按键后，动画区在 300ms 内开始响应，全程跟随输入节奏 |
| **等号深度展开** | 按下 `=` 后，动画自动播放完整计算链；用户可点击暂停并单步执行 |
| **Hover 即解释** | 鼠标悬停在任意绿色方块/寄存器/地址/图形点上，浮窗显示其当前值与语义 |
| **架构对比模式** | 支持左右分屏对比同一表达式在 ARM64 vs x86-64 下的执行差异 |
| **表达式快照** | 可将某次计算的完整动画导出为可分享的交互式快照或视频片段 |
| **图形交互** | 2D 模式下滚轮缩放、拖拽平移；3D 模式下右键旋转、中键平移、滚轮缩放 |
| **参数联动** | 拖动滑块时，所有依赖该参数的表达式实时更新，轨迹模式保留历史帧 |
| **逐点求值** | 在函数图像上点击任意点，弹出该点的完整求值过程动画（AST + 数值流） |

### 5.3 视觉风格

- **色彩体系**：
  - 背景：极深色（#0a0a0a 至 #121212），营造终端/底层感。
  - 强调色：科技绿（#00ff41 / #0aff0a）用于数据流、方块、指针连线。
  - 辅助色：冷灰（#8b949e）用于静态结构、注释；琥珀色（#ffa657）用于警告/进位/溢出指示。
  - 图形色彩：每条表达式分配唯一的鲜明色彩（遵循色盲友好调色板），支持用户自定义。
- **字体**：等宽字体（Monospace）用于地址、寄存器名、二进制显示、代码表达式；无衬线字体用于 UI 标签；LaTeX 风格字体用于数学公式渲染。
- **方块美学**：所有内存单元、寄存器、链表节点统一为圆角矩形方块，保持 3B1B 式的几何纯粹感。
- **动画质感**：缓动函数以 `ease-out-expo` 为主，数据流动用连续贝塞尔曲线，避免生硬跳跃。
- **坐标系风格**：网格线使用极细半透明线（#1f1f1f），坐标轴使用白色，刻度标签使用冷灰色。

## 6. 非功能需求

### 6.1 性能
- 动画帧率：核心可视化区在标准设备上稳定 **60 FPS**。
- 响应延迟：从按键到首帧动画开始 ≤ **100ms**；复杂表达式（如多层阶乘）的完整动画预计算 ≤ **500ms**。
- 图形渲染：2D 函数图像在 1080p 视口下采样与渲染 ≤ **16ms**；3D 曲面网格（10k 顶点）渲染 ≤ **16ms**。
- 内存占用：运行时内存峰值控制在合理范围，支持千步级操作历史回溯；3D 网格数据支持流式加载。

### 6.2 精度与正确性
- 数值计算结果必须与标准 IEEE 754 双精度浮点及 64-bit 整数运算严格一致。
- 可视化展示的是**真实计算过程的抽象动画**，而非装饰性特效；位运算、栈操作、寄存器转移必须与所选架构的抽象模型逻辑自洽。
- 图形采样必须忠实于数学定义，自适应采样算法的误差可控；隐函数渲染的 Marching Cubes 算法必须正确提取拓扑结构。

### 6.3 平台无关性
- **渲染层**：基于标准图形 API（如 WebGPU / Vulkan / Metal / DirectX 的抽象层）或高性能 2D Canvas，确保跨平台一致性。图形模式优先使用 GPU 加速的渲染管线。
- **计算层**：核心数值引擎使用平台无关的定点/浮点算法，不依赖特定硬件指令集实现功能正确性（可视化层才体现架构差异）。
- **输入层**：自适应键盘（物理键盘、触屏、游戏手柄等）。

### 6.4 可扩展性
- 架构插件化：新增模拟架构（如 MIPS、LoongArch）仅需实现寄存器模型与指令映射，动画引擎复用。
- 可视化插件化：支持第三方定义新的“层级”（如 L9: 缓存行/Cache Line 可视化）。
- 图形插件化：支持第三方定义新的采样器、坐标系、后处理效果（如曲率热力度、向量场 LIC 纹理）。

## 7. 里程碑规划

| 阶段 | 目标 | 关键交付 |
|------|------|---------|
| **M1: 内核** | 完成高精度 64-bit 计算引擎与表达式解析器 | 支持全部图中计算模式，结果正确 |
| **M2: 骨架** | 完成可视化引擎基础框架 | 方块系统、连线系统、寄存器/内存网格、时间轴控制 |
| **M3: 链路** | 打通“按键 → 内存 → ALU → 结果”完整动画链路 | 标准/科学模式的可视化 |
| **M4: 硬核** | 实现程序员模式深度可视化与多架构模拟 | 位运算、进制、RISC-V/ARM64/x86-64 切换 |
| **M5: 图形基座** | 实现图形模式的 2D 渲染引擎与表达式系统 | 显函数、参数曲线、极坐标、不等式、点线渲染 |
| **M6: 图形进阶** | 实现 3D 渲染引擎与高级数学对象 | 参数曲面、隐函数、向量场、流形可视化 |
| **M7: 计算可视化** | 打通图形模式下的计算过程可视化 | AST 动画、采样过程、求值链路、积分近似动画 |
| **M8: 参数动画** | 实现参数驱动与动画导出 | 滑块系统、轨迹追踪、关键帧、视频导出 |
| **M9: 抛光** | 3B1B 级动画质感调优、性能优化、跨平台适配 | 公开发布 |

## 8. 成功指标

1. **教育价值**：用户能仅凭 Calcore 的动画，向他人解释清楚“为什么 0.1 + 0.2 ≠ 0.3”（浮点精度）、“递归阶乘时栈里发生了什么”、“参数方程如何映射为曲线”、“Marching Cubes 如何提取隐式曲面”。

4. **数学表达力**：Calcore 能够完整渲染标准微积分/线性代数/微分几何课程中的全部经典函数、曲面与流形示例。

### 9.1 术语表

| 术语 | 说明 |
|------|------|
| **Manim 引擎美学** | 指 3Blue1Brown 视频中标志性的几何化、符号化、流畅数学动画风格 |
| **绿色实心/空心方块** | Calcore 的核心视觉隐喻，分别代表数据实体与指针/空地址 |
| **指针段位移拼接** | 模拟 x86 等架构下逻辑地址 = 段基址 + 偏移地址的硬件寻址过程 |
| **调度场算法 (Shunting Yard)** | 将中缀表达式转为后缀表达式（RPN）的经典算法，用于展示运算符栈行为 |
| **Marching Cubes** | 从三维标量场中提取等值面的经典算法，基于体素网格的 256 种拓扑配置 |
| **参数化图卡 (Chart)** | 微分几何术语，指流形上的一个局部坐标映射 `φ: U → M`，图卡集构成 Atlas |
| **Frenet 标架** | 由切向量 T、法向量 N、副法向量 B 构成的沿曲线的局部正交坐标系 |
| **Transition Map** | 流形上两个图卡重叠区域之间的坐标变换映射，展示流形的局部拼接结构 |
| **LIC (Line Integral Convolution)** | 向量场可视化技术，通过沿流线卷积噪声纹理来呈现场的流动感 |
